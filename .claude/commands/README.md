# Custom Claude Commands for Product Managers

> **Supercharge your AI-assisted workflow with battle-tested Claude Code commands for Product Managers.**

This collection provides 19 ready-to-use custom commands for Product Managers that turn Claude Code into your expert PM and engineering assistant.

## Why Use Claude Code Commands?
- One-word trigger (e.g., `/prd`, `/competitive`, `/mockup`) gets repeatable action by Claude Code
- Specify the file naming convention, file location once and file gets made and put there
- Specify the MCP conventions, how you want a lead categorized or a ticket created and it happens
- Specify a template for how you want an output to look so you get a consistent format and sections to your documents
- Consistent, high-quality outputs using proven frameworks
- Customisable to your team's specific needs

## What's Inside
Markdown documents of 20 Custom Claude Code commands that we have been using and that you can modify as you wish for your business and use case. Each command provides structured prompts, best practices, and frameworks so Claude understands your context and delivers exactly what you need.

## Tailor These Commands To You and Your Org 
- These commands are markdown documents. Edit them as much as you wish. Specify what file location Claude should put them in, what competitors to compare against, etc..
- Write your own custom commands. Have Claude Code help you.

## Complete Command List
### Identify Your Next Feature
- `/plan` - Create structured plan documents specifying the problem to solve and the goal using frameworks like Jobs to Be Done

### Research the Market
- `/competitive` - Conduct competitive analysis, compare how competitors handle specific problems, and document competitor UI
- `/customer-interview-simulate` - Simulate customer interviews with customers in your Ideal Customer Profile
- `/customer-interview` - Prepare and conduct real customer interviews
- `/research` - Synthesize insights from large datasets, review user feedback, and analyze social media discussions

### Create a Feature Plan
- `/plan` - Build feature plans with the problem to be solved, job to be done, success criteria, risks, and open questions
- `/prd` - Create comprehensive Product Requirements Documents according to corporate format

### Create a Mockup
- `/mockup` - Create UI mockups by examining existing code, stylesheets, and design patterns

### Get Informed on Technical Issues
- `/understand-feature` - Examine the codebase and explain how features work technically, key technical issues, and implementation details
- `/edge-cases` - Identify edge cases and error states that should be considered during implementation

### Submit to JIRA / Linear
- `/status` - Create status updates and manage tickets in Linear/JIRA

### Understand Coding Progress
- `/github-status` - Summarise GitHub progress, recent activity, and PR significance

### Investigate and Submit a Bug
- `/bug-report` - Document bugs with questions, code investigation, and submit to Linear/JIRA

### Launch and Sales Communications
- `/launch` - Write product launch announcements based on plans
- `/sales-enablement` - Create sales battlecards and enablement materials
- `/documentation` - Generate technical documentation and API specs

### Handle Customer Requests and Feedback
- `/triage-requests` - Turn Discord messages, survey responses into separate bugs or feature requests, check against existing Linear tickets
- `/feedback-analyze` - Analyze customer feedback patterns and extract insights

### Additional Commands
- `/strategy` - Draft strategy memos with analysis and recommendations
